我和我的软件们：日记（02）

发表于 2020-03-28 更新于 2020-04-30 分类于 软件

前言

本期来说一说日记软件。

真正意义上的日记软件，我在这里只推荐一个，所以本期主要并不是来推荐软件，而是想说说我对这种相对比较小众和专有功能的软件的看法。

日记软件需要什么

通用格式导入导出

什么是通用格式呢？可以简单理解为这个格式中所包含的内容可以被主流软件读取，里面所包含的内容可以被人类理解。

这段文字就是上面内容的经过加密后的文本，可你能看出来我写的是什么内容吗？答案当然是否定的。

大多数软件通常很难长久的更新下去，随着新系统的发布，技术的更新，又或者是软件开发者的自身原因，总会有新软件的兴起，旧软件的衰败。一旦一个软件停止维护，随着系统更新、手机升级，将产生越来越多的 Bug；甚至因为软件使用的某种开发方案不再被新系统支持，将直接导致软件无法在新系统上工作。

如果您是一个真正的日记爱好者，那您一定想在未来十几年，甚至几十年之后，看到我那时候都在做什么，想什么。但是，当您在新系统上安装上那个已经数年未更新的软件之后，发现软件已经无法工作，仅有旧手机上还保留着原先的日记。您想把数据导出来存档，却发现软件并不支持导出，或者导出的数据都是加密后的，根本无法被读取和阅读，那就只能让这些日记永远尘封在旧手机上了。

如果这款已经停止更新的软件可以以通用格式导出，而您准备用的新日记软件又支持通用格式导入的话，那么您的日记迁移便可以做到无缝衔接，或者仅需要稍作修改。

上面说的情况其实不是个案，生活中数量可观的软件都不提供导出功能，甚至一部分老式手机的通讯录、短信也无法导出，所带来的不方便，相信大家也是体会过的。这也更凸显了笔记（日记）类软件，以 通用格式导入导出 的重要性。

友好的记录功能

我认为一个日记软件应当可以如下元素：

  • 主体：文字
  • 媒体：图片、语音、视频
  • 环境：时间、地点、天气

当然，如果您还把日记当成游记，或者是手账本用的话，软件还应当支持更强大的图文混排，如 markdown，或者是类似 onenote 的无限扩展画布。很多日记软件只能记录几句话和日期，那这样的软件和备忘录又有什么区别呢？

那年今日

几年下来，我记录了有数千条日记，但是真的想翻一翻日记，要从头刷到尾也是很困难的，所以 “那年今日” 是我觉得一个相对比较重要的功能。几乎每一天，我都可以看到去年的今天我在做什么，前年的今天我在做什么…… 时光流转，物是人非，“那年今日”能让人有一种穿越时空的感觉，也能带你找回当年的记忆。

前段时间，我用 QzoneExporter 导出了 QQ 空间所有说说，并导入了我接下来将会推荐的软件 Day One。我只要打开 Day One，就可以看到几年前的今天，我发了什么动态，想来还是别有一番滋味的。

真的需要日记软件？

上面所说的一些功能仅仅是我个人的看法，如果你对这些需求并不感冒，那么就需要考量一下自己是否真的需要一个专业的日记软件了。

回归日记的本源，无非是时间地点人物事件，比如：

【手动狗头】你们看，完成一篇日记需要一个专业软件吗？答案是否定的。我也曾经陷入这样的误区，觉得什么需要都要搞一个对应的软件来做，结果增加了学习和经济成本，却没有带来对应的效率提升，希望大家可以不要陷入这样的误区。

某种程度上来说，你手上任何一个可以写文字和图片的软件都可以完成记日记的需求，小到记事本，大到Word，等等。只是有些功能可能需要手动实现，有些功能用起来没有那么顺手罢了。但是比起多使用一个软件，多增加一份成本，这个问题并非不能克服。

软件推荐

我和我的软件们：前言（01）

发表于 2020-03-28 更新于 2020-04-30 分类于 软件

引言

鉴于 App Store 内软件质量良莠不齐，很多软件过度关注新颖、美观和价格，却做不到此类软件最应该做到的功能，这样的软件并不具有过高的实用性，如某日记软件每天只能记一篇日记，看似高端小清新，实则实用性全无。

软件之所以为软件，首先是工具，其次才是艺术品，不具有良好的稳定性和可扩展性，再漂亮的软件也终究只是花瓶而已。

综上，本专题将对本人使用过的软件进行主观性评测，评测仅代表个人意见，无任何软件厂商对本人提供任何形式的赞助。

介绍

因为学业原因，我打算用一年半的时间写完这个系列，目前计划的内容如下：

  • 日记
  • 笔记
  • 阅读（器）
  • 存储
  • 效率

每款推荐软件将包含如下信息：

  • 平台
  • 价格
  • 软件版本
  • 这类软件应当做到哪些
  • 软件的优点、劣势
  • 我是怎么用这个软件的
  • 如何和其他软件联动的

软件抛开使用途径来谈好坏没有意义，因此我会从数据创造到数据录入这一完整链条来说明软件使用。

如日记篇，我将会提到我用“微信”进行语音转换构建日记的文本，用“NOMO”拍摄“每日一记”的照片构建日记的图片；以及利用 “IFTTT”，从“instagram”保存我发布照片，从“YouTube”保存我喜欢的视频等等。

本系列主要针对 iOS、iPadOS、macOS，未来也可能会提到 watchOS、Windows 等其他平台的效率软件。

正版声明

本人不使用任何定价针对个人（如 AutoCAD 除外）销售的盗版软件，所有推荐软件均为自购正版，并在 TOS (Terms of Service) 框架内使用。

我的设备

  • MacBook Air (Early 2015, 256G)
  • iPhone 8 (256G)
  • iPad mini 5 (64G) + Apple Pencil
  • 鼠标：Logitech M590
  • 键盘：ANNE PRO 2

Apple 软件付费

Apple 的软件主要分为在商店购买和内购两种，具体如上图所示。

一般来说，商店购买获得的是软件的所有权，让你对软件的界面、交互有所了解，并体验一部分功能；而内购则属于软件的增值服务，可用于去除广告，解锁某项或全部的功能等。

通常，软件所提供的价格为当前软件版本，当前系统版本下的价格。一次性买断软件一般提供小版本的免费更新：某软件由 v1.1 升级至 v1.2，则应当免费升级；而由于各种原因（如系统更新等），需要从 v1.2 升级至 v2.0 这样的大版本更新，则有可能需要重新付费或者更换付费策略，这就要看软件厂商的升级方案了。

订阅制软件一般允许用户直接更新至最新版本，需要注意的是，大部分订阅软件会提供免费试用，但根据苹果的扣费策略，在你进行扣费授权之后，所有的账单将由苹果接管，到期自动扣费，不会提醒。

如何取消订阅

SETAPP

Setapp 是著名的 macOS 正版软件订阅模式提供商，只需要一笔月使用费，就能随意使用其库内的 macOS 软件，相当于一个软件工具集， 包括 iStat Menus、Bartender、CleanMyMac X、Ulysses、BetterZip、iMazing、MoneyWiz、XMind 等等在内的近 200 款 macOS 软件。

Setapp 的年费为 $8.99 * 12月，折合人民币 770 元左右，双设备使用；本人使用的为家庭版单人授权（Setapp 允许非家庭成员合租），价格约 210 元，单设备使用。

需要注意的是，不同厂商虽然都有“家庭版”的购买方案，但是对“家庭”的定义有所不同。某些厂家支持非家庭成员组成“云家庭”合租，而有些厂商则要求家庭组必须由家庭成员组成。

对订阅制软件的看法

对于存在云服务（存储）并且（或）使用收费接口并且（或）需要长期稳定维护的生产力软件，订阅制是完全可行的，如云盘，笔记，文字识别（翻译）软件等；但也有存在一些软件滥用订阅制，则需要多加鉴别了。

对老牌软件的看法

不得不说，尤其是一些核心生产力软件，老牌就是老牌。比如同价位下，目前还没有见过几款 PDF 阅读编辑器可以干过 PDF Expert，没有几款手写笔记软件可以干过 GoodNotes，Notability 等等。

老牌软件所带来的并不是惊艳感，而是功能足够，性能稳定，长期使用下来会给你一种感觉：就这个软件，够用，不换了。

写在结尾

  1. 工具是用来帮助我的，要做工具的主人而不是工具的奴隶。
  2. 免费的才是最贵的，不要指望有又免费又好用，还能长期维护的工具，用爱发电并不能喂饱开发者。

获取 iOS 设备所链接 Wi-Fi 的具体信息

发表于 2020-03-19 更新于 2020-03-24 分类于 教程

前言

很多人像我一样，在测试 Wi-Fi 的时候需要知道我的设备链接到 Wi-Fi 的什么频段，什么信道，信号强度如何等等，但 iOS/iPadOS 并不像 macOS 一样，对用户直接提供 Wi-Fi 的详细调试信息【macOS 可通过按住 option，鼠标点击 Wi-Fi 图标来获取】。因此，我们可以使用 Apple 面向开发者的调试文件来获得。

下载

  1. 进入 https://developer.apple.com/bug-reporting/profiles-and-logs/，找到 Wi-Fi for iOS 的字样，并下载后面的 Profile 文件；
  2. 将上述配置文件传输至 iPhone/iPad，当然你也可以直接在手机上直接操作上述内容。

安装

  1. 点击“设置” -> “通用” -> “描述文件”，找到刚刚导入的描述文件；
  2. 点击右上角“安装”，并输入密码完成安装。

使用

  • 进入 Wi-Fi 设置界面，打开当前所连接 Wi-Fi 的详细信息，选择“Diagnostics”，就可以看到 Wi-Fi 的详细信息啦。

备注

  • 为了避免一些不必要的系统日志或者引发一些 Bug，个人不建议在非调试需求下长期打开调试模式；如需关闭，请删除描述文件。
